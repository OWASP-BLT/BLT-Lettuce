Discovery Paths Explained

Technology-Based Path

  1. 1 Ask Technology: Which stack? Python, Java, JS, Mobile, Cloud Native, etc.
  2. 2 Ask Difficulty: Beginner, Intermediate, or Advanced
  3. 3 Ask Project Type: Tools, Code Repos, Docs, Training
  4. 4 Filter Metadata: Filter by tags, type, level, pitch, and activity
  5. 5 Rank Results: By relevance, freshness, popularity, and beginner-friendliness
  6. 6 Recommend Projects: Top 3 matching open-source projects

Mission-Based Path

  1. 1 Ask Mission: Learn AppSec, Contribute Code, Documentation, GSoC Prep, Research, etc.
  2. 2 Ask Contribution Type: Code, Documentation, Design, Research, etc.
  3. 3 Filter Metadata: Match type, tags to mission keywords, maturity level, training tags
  4. 4 Rank Results: By mission relevance, activity, maturity, and pitch quality
  5. 5 Recommend Projects: Dynamic list based on the user's goal

Fallback Logic (Low Confidence)

  • Expand similar tags
  • Show popular community projects
  • Suggest Beginner Training resources
  • Offer "Try Another Filter" option
