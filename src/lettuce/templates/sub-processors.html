Sub-Processors

Last Updated: March 22, 2026

Overview

BLT-Lettuce uses third-party sub-processors to deliver its services. This page provides comprehensive information about each sub-processor, including their role, data handling practices, and compliance commitments.

By installing or using the Bot, you consent to data processing by these sub-processors as described below. We maintain current information about all sub-processors and notify users of material changes.

Sub-Processor Directory

1. Cloudflare, Inc.

Primary Role

Cloudflare is the primary infrastructure provider for BLT-Lettuce, hosting all application logic, databases, and data processing.

Services Provided

Service Purpose Data Processed
Cloudflare Workers Serverless computing platform executing Bot logic Webhook payloads, API requests, user messages, configuration data
Cloudflare D1 Managed SQLite database for persistent data storage Workspace data, channel information, user profiles, event logs
Cloudflare Security DDoS protection, WAF, rate limiting, encryption Network traffic, IP addresses, request metadata
Global Network Content delivery and failover across data centers All customer data subject to replication and redundancy

Data Centers

Cloudflare operates data centers in multiple regions worldwide:

  • United States – Multiple regions for US-based workspaces
  • Europe – GDPR-compliant regions for EU data subjects
  • Global – Additional regions as needed for performance and compliance

Data Security Practices

  • Encryption: Data encrypted at rest using AES-256; in-transit using TLS 1.3
  • Access Controls: Role-based access control (RBAC), principle of least privilege
  • Redundancy: Multi-region replication for disaster recovery
  • Monitoring: 24/7 security monitoring and incident response
  • Compliance: SOC 2 Type II, ISO 27001, GDPR, CCPA

Data Retention

Cloudflare retains customer data for the duration of service. Backup copies may be retained for up to 90 days after deletion for disaster recovery purposes.

Privacy & Compliance

For detailed information about Cloudflare's data processing practices, privacy policies, and compliance certifications, visit:

Cloudflare Privacy Policy Cloudflare Compliance Cloudflare Trust Hub
2. Slack Technologies, Inc.

Primary Role

Slack is the platform through which users interact with BLT-Lettuce. The Bot operates within Slack's ecosystem using their APIs and event infrastructure.

Services Provided

Service Purpose Data Processed
Slack API Bot communication with Slack workspaces Messages, user information, channel metadata
Event Subscriptions webhooks for channel joins and other events Event payloads including user IDs, channel IDs, timestamps
OAuth 2.0 Bot installation and workspace authentication Bot tokens, workspace credentials, installer information
Workspace Administration Workspace data access and configuration Team ID, workspace name, member counts

Data Processing

Slack processes data according to their terms of service and privacy policies. BLT-Lettuce only requests the minimal scopes necessary for Bot operation:

  • app_mentions:read – Detect mentions for engagement
  • channels:history – Read channel conversation history for context
  • chat:write – Send messages to channels
  • commands – Handle slash commands
  • users:read – Access user information for personalization
  • team:read – Access workspace information

Data Security

Communication with Slack APIs uses industry-standard TLS encryption. Slack maintains enterprise-grade security practices.

Privacy & Compliance

For information about Slack's data handling practices:

Slack Privacy Policy Slack Trust & Compliance Slack Security

Data Flow Summary

Customer data flows through the following processors:

  • User joins channel in Slack → Slack generates event webhook
  • Event transmitted to Cloudflare Workers → Bot logic processes event
  • Bot logic queries Cloudflare D1 database → Retrieves workspace configuration
  • Bot sends response back to Slack → Message delivered to user via Slack API
  • Event logged in Cloudflare D1 → Audit trail stored for compliance

Sub-Processor Agreements

We maintain Data Processing Agreements (DPAs) or equivalent agreements with all sub-processors to ensure compliance with data protection regulations including GDPR and CCPA.

Changes to Sub-Processors

We may add new sub-processors or replace existing ones as the Bot evolves. We will:

  • Maintain a current list of all active sub-processors
  • Notify users of material changes to sub-processor arrangements
  • Ensure all sub-processors maintain equivalent data protection standards
  • Provide users with options to object to changes where legally permissible

GDPR Compliance

Under GDPR Article 28, we ensure that:

  • All sub-processors process data only on our documented instructions
  • Sub-processors maintain appropriate technical and organizational measures
  • Sub-processors are bound by confidentiality obligations
  • We have assessed sub-processor capacity to ensure GDPR compliance
  • Sub-processors provide sufficient guarantees for data protection

Contact & Questions

For questions about sub-processors, data handling, or to request information about how your data is processed:

  • Review the individual sub-processor privacy policies linked above
  • Contact the Bot maintainers through the official repository
  • Submit data protection inquiries through your workspace administrator
