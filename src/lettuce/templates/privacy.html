Privacy Policy

Last updated: March 22, 2026

What BLT-Lettuce Collects

BLT-Lettuce stores limited workspace and bot-operation data required to provide bot features, such as workspace metadata, channel metadata, repository metadata connected by administrators, and activity events needed for statistics and operational health.

Slack Data Handling

When installed, BLT-Lettuce may process Slack user IDs, channel IDs, team IDs, and message event metadata that are required to deliver welcome messages, commands, and analytics. Data processing occurs through approved Slack APIs and configured bot scopes.

How Data Is Used

Collected data is used to power onboarding messages, repository discovery, operational dashboards, workspace-level metrics, and reliability features such as inactivity alerts and error monitoring.

Data Storage

Data is stored in Cloudflare-managed services (including D1 and Worker runtime resources) according to deployment configuration. Workspace administrators can request removal by disconnecting the bot from a workspace using supported administrative controls.

Third-Party Services

BLT-Lettuce integrates with Slack APIs and may integrate with GitHub APIs and Sentry for error reporting, depending on deployment settings. These services are subject to their own privacy terms.

Security

BLT-Lettuce uses signed request verification and deployment-level secret management. Administrators are responsible for secure deployment configuration, least-privilege permissions, and access controls for infrastructure resources.

Contact

For privacy questions related to this project, open an issue in the project repository: OWASP-BLT/BLT-Lettuce Issues.
